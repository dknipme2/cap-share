CAP · Honest assessment

What's built, what's missing, and how we compare

The story pages are the outward-facing view. This is the inward-facing one. Where does CAP actually sit today? What does it do well, where does it strain, and what does it have to become to serve more than the current scale point — single-company embed with two or three analysts and many consumers.

Internal framing · living document. Updated as capability lands. Named gaps, phased plan, and cross-platform comparison are point-in-time reads — revisit quarterly or when any Phase 1 / 2 item closes.

Where we are

The current scale point is real — and narrow

CAP is proven at one company, two-to-three analysts, many consumers. Everything under that bar works well. Above it, the coordination and permissioning primitives that let the small team fly start to strain.

Works today

1 company · 2–3 analysts

Single-tenant embed. Worktree isolation + .active markers + CODEOWNERS + personal sibling repos carry the whole coordination load. Disciplined, works cleanly.

Works today

Many consumers

claude.ai Projects serve managers and VPs at arbitrary scale. Read-only graph access, evidence-traced follow-ups, no IT install. Scales linearly without system-side pressure.

Strains above bar

5+ analysts, multi-tenant, regulated

Coordination becomes overhead. Permissioning is catalog-level + CODEOWNERS, not per-engagement or per-entity. No SSO, no enterprise audit service, no formal governance workflow.

What's built

The capability inventory — real, running, load-bearing

Rules stack + graduation

  • Feasibility engine (tools/feasibility_check.py) — analytical roles × grain match against method YAMLs

Multi-instance coordination (small scale)

  • Worktree isolation per analyst session
  • .active markers with file-overlap warnings
  • Personal sibling-repo pattern for analyst workspaces
  • Scope signaling (business / personal / unscoped)
  • Session YAML append-only log

Consumer surface

  • claude.ai Projects for manager / VP consumption
  • Evidence-traced follow-ups via graph queries
  • Share publish surface (this site) with strict IP-scrub contract
  • Session-bridge prompts for multi-session handoff

Dashboarding · static + interactive

  • Static: self-contained HTML dashboards (D3.js inline, no build step); the share/ surface + GitHub Pages auto-deploy for board and meeting artifacts
  • Interactive: claude.ai Project boards with live KPIs, bar charts, status stripes, and a cursor — the "board you can talk to"
  • Storyboards: scenario-based artifacts (portfolio review, launch readiness, LOE defense) — board view paired with conversational drill-down
  • PPT deliverables: template-based python-pptx for CAP-branded slide decks when the consumer workflow is deck-based
  • No IT install — runs in the tools stakeholders already use

Guardrailed free-query experience

  • Every free-form question answered against the graph, not the model's memory — answers stitched from structured entities with stable IDs
  • Evidence trace attached to every answer (finding · method · source · confidence tier)
  • Confidence-flag system (7 tiers) surfaces unverified claims rather than hiding them — [CONFIRMED] / [STRUCTURAL] / [DIRECTIONAL] / [CALCULABLE] / [SPECIFIC] / [STALE] / [ESTIMATED]
  • Cross-engagement recall: prior decisions / outcomes persist and feed current answers automatically
  • Hallucination defense: claims without graph grounding land as [SPECIFIC] or [ESTIMATED], not as facts

What's missing

The honest gap inventory · enterprise readiness

Named in plain language. No euphemisms. Each gap has a severity tag — blocking at enterprise scale, caution where a workaround exists, future where the shape is known but the trigger hasn't fired.

Gap · collisions

Concurrent-analyst coordination above ~3

Today: .active markers warn on file overlap between concurrent sessions. Worktrees isolate working state. At 5–10+ concurrent analysts, markers become noisy and overlap becomes frequent. No merge-queue, no lock service, no conflict-auto-resolve. Two analysts editing the same entity YAML still requires a human rebase.

BLOCKING · scale
Gap · permissions

Per-engagement and per-entity permissioning

Today: catalog-level (cdp_dev vs cdp), CODEOWNERS on sensitive paths, personal repos for isolation. No per-entity row-level access. No per-engagement "analyst A can see Cycle entities but not Client-B entities." No enforceable graph-layer permissions. Fine at one company; breaks at multi-tenant.

BLOCKING · multi-tenant
Gap · authN / SSO

Authentication is git identity + machine auth

No SSO, no directory integration, no user-provisioning workflow. Access to the repo, Databricks catalogs, and claude.ai Projects is provisioned separately and manually. Enterprise IT won't approve this pattern at scale.

BLOCKING · enterprise
Gap · audit trail

Git log + session YAMLs are the audit surface

Comprehensive for 2–3 analysts. Not regulator-grade. No centralized audit log service, no tamper-evident storage, no structured events for "who read what, when." HIPAA / SOC-2 audit responses would be manual forensic work.

CAUTION · regulated use
Gap · multi-tenancy

Single-tenant per deployment

The domain layer (cap-base-domain.md) is swappable, but per-tenant would require duplicate infrastructure today. No tenant-routing, no per-tenant KB indexes, no cross-tenant IP isolation enforced by the system. Each new company = new CAP instance.

FUTURE · SaaS shape
Gap · governance workflow

Heuristic graduation is informal

Today: memory → heuristic register → triad review → rule file. Triad is informal (dispatched from a session); no queue, no formal approval, no role-based sign-off. Works at single-analyst pace; would need workflow enforcement at team scale.

CAUTION · team scale
Gap · uptime / DR

No SLAs, no DR plan beyond git remote

GitHub is the canonical store; Databricks is the compute layer. Both have their own DR. CAP itself has no "platform availability" story — if GitHub is down, sessions can still proceed in local state, but no centralized monitoring exists. Fine for embedded analyst work; not shippable as "platform uptime".

FUTURE · SaaS shape
Gap · onboarding

New-analyst onboarding is manual + disciplined

Today: clone repo, read CLAUDE.md, skim last 3 sessions, scope-signal in .active marker. Works because the analyst cohort is tiny. No curriculum, no gated onboarding, no automated skill-assessment. New analysts learn by pairing.

CAUTION · team scale
Gap · enterprise search

KB search is analyst-only

tools/kb_index.py runs locally during sessions. No shared search surface for managers/VPs outside claude.ai Projects. No "cross-engagement search" for discovering prior analogs automatically across companies.

FUTURE · consumer expansion
Gap · observability

No system-level telemetry

Session YAMLs capture engagement-level activity; hooks log enforcement events. No unified dashboard for "how is the capability performing across engagements" — throughput, pattern reuse rate, compound-curve position per question family. Would need to stand up to prove the compounding thesis at scale.

The positional read

Foundry is the nearest neighbor on the graph + enterprise-readiness axes. It has the ontology and the compliance shape; it lacks the lightweight methodology+graduation loop and the agent-native skill system. Switching cost is high; consulting cost is high.

Backstage is the nearest neighbor on the typed-graph-as-OSS-pattern. It proves the typed-entity pattern works at enterprise scale. It doesn't do methodology or agents. CAP borrowed the pattern; not the product.

Claude Projects + connectors is the nearest neighbor on the consumer surface. Useful for managers as a front-end; it's not a platform. CAP uses it as the consumer skin over the real capability underneath.

CAP's distinctive position: typed entity graph + methodology/graduation + agent skill system + KB retrieval fused together, tuned to a single commercial domain, portable by config swap. Narrow today. Real today.

The shape of what's next

Phases, not promises

Dated commitments are a different artifact. What follows is the dependency order — which gap has to close before which becomes tractable.

PHASE 0 · today Single-company embed at 2–3 analysts

Proven. The capability runs. Analyst throughput is real; manager-facing consumption via claude.ai is scaling linearly. Compounding curve is demonstrable on repeated engagements of the same family.

Covers. Single company, analyst team ≤ 3, many consumers (managers/VPs), governed by git + CODEOWNERS + personal repos.

PHASE 1 · next Multi-analyst coordination + governance workflow

What has to close. Concurrent-analyst coordination above the .active-marker ceiling. Formal queue for heuristic graduation (not just session-dispatched triads). Per-engagement permissions primitive — at minimum, scoped-read within the graph.

What it unlocks. 5–10 analyst team at one company. Formal methodology governance. Measurable throughput dashboard.

PHASE 2 · future Enterprise IT alignment (SSO / audit / DR)

What has to close. SSO + directory integration. Audit service (not git log). DR plan + platform-level uptime story. Compliance-audit response shape.

What it unlocks. Acceptable to enterprise IT. Deployable in regulated environments (pharma compliance, financial services analog).

PHASE 3 · future Multi-tenant SaaS shape

What has to close. Per-tenant isolation enforced by the system, not by swap-file discipline. Cross-tenant IP separation. Tenant-routing at KB + graph layer. Enterprise-search for managers across tenant boundary (where permitted).

What it unlocks. Deployable as a managed platform across multiple companies from a single vendor instance. A real SaaS shape.

PHASE 4 · future Domain extension + cross-domain benchmarks

What has to close. Second domain proof (e.g., manufacturing, insurance, energy). Cross-domain pattern library (which methodologies port across, which don't). Benchmark registry across companies (opt-in).

What it unlocks. CAP becomes a multi-domain capability. The "agent is the product; domain is config" thesis is proven by repetition.

The line to take with a VP asking "is it ready?"

Ready for your use case, not ready for every use case.

If you have a single company, 1–3 analysts, and an ambition to stop losing the reasoning after every engagement: CAP is ready today, and the compounding curve starts on engagement one.

If you have a larger analyst team, regulated handling, or multi-tenant ambitions: CAP is not there yet. The gaps above are named with severities; the phase plan is real; the investment shape is knowable.

What you get today either way: a narrow but deep capability, with an honest gap list, that runs at production quality on the scale point it was designed for.
