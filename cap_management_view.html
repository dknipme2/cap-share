How the plumbing actually works

What makes the conversation possible

The reason the manager can drill three layers deep and still get grounded answers — instead of confident hallucination — is that the conversation runs against the graph, not against the model's memory.

01 · THE INTERFACE

claude.ai Projects with CAP connector

Manager opens a Project on claude.ai scoped to their engagement. The project is preloaded with entity summaries, current KPIs, and the methodology scope. No IT install, no new tool — the interface they already use.

02 · THE GROUND

Entity graph + KB in the background

Every follow-up triggers a graph query — this asset, this method, this finding. The model stitches the answer from structured sources instead of making it up. The graph is shared: the same asset referenced in the board is referenced in the answer, with the same ID.

03 · THE EVIDENCE

Trace attached to every answer

Each answer carries a trace block: which finding, which method, which assumption, which source table. The manager can click one level deeper and see the underlying record. If they want to challenge the answer, they have the receipts in the next click.

The key distinction

Who builds vs. who consumes

Two roles, two interfaces. Both Claude.

The analyst · builds
  • Forge + Atlas for deep analytical work
  • Subagents, triad review, methodology selection
  • Writes entities, briefs, methods, recommendations
  • Outputs: the board, the storyboard, the underlying evidence
  • Context: the full CAP system map
The manager · consumes
  • claude.ai Projects — the interface they already use
  • Dashboard / storyboard as the starting artifact
  • Follow-up questions drill through the graph
  • Decisions captured back to the graph
  • Context: the engagement scope, not the plumbing
