CAP · Why the layers exist

Claude is smart. It isn't your analyst.

Any large language model answers the question in front of it — often well. What it can't do out-of-the-box is remember last month's decision, know your portfolio, follow your methodology, challenge its own answers, or learn from being used. CAP is the specific plumbing that closes each of those gaps, layer by layer. The model is the engine; these layers are what make it a commercial analyst.

Six things out-of-the-box Claude can't do

And the layer that fixes each one

Each row is a real gap. Without the corresponding layer, the answer comes back confident and plausible — and probably wrong in a way that only a domain expert would catch.

It can't · by default

Remember what happened last engagement

Every chat starts fresh. The decisions, methodology tuning, and context from last month's FF sizing project live in someone's head or a stale deck — not in the model's working memory.

→
CAP adds

Persistent memory + session YAMLs

An auto-memory layer (feedback, user profile, projects, references) plus an append-only session log. Every session starts by reading the last three. What was decided, why, and by whom — all recoverable.

It can't · by default

Know your commercial architecture

It doesn't know which assets you sell, which therapy areas you're in, which vendors you source data from, which payers matter. Asking the model about your portfolio returns generic industry takes.

→
CAP adds

Typed entity graph + domain KB

Organization, product, therapy-area, market, event, and data-source entities captured as structured YAMLs. Hybrid retrieval with cross-encoder rerank over a typed knowledge base. The model reasons over your commercial architecture, not "pharma in general".

knowledge/graph/ + knowledge/ (KB)
It can't · by default

Follow your firm's methodology

Asked to size a field force, a raw model picks a method from whatever's in its training data. It doesn't know which method you use for specialty launches vs primary-care, which decision tree routes the question, or where the validated method config lives.

→
CAP adds

Methodology taxonomy + role skills

A decision tree classifies the question shape. Role skills (strategist, operator, researcher, analyst) load the right procedures. Method YAMLs constrain the analysis. Every method choice is traced to the decision tree, not improvised.

L5 skills + methodology/
It can't · by default

Enforce your conventions mechanically

Reminders in a system prompt get forgotten, ignored, or papered over when the model is under pressure to finish. "Always cite sources" is a soft request; models drift from it silently.

→
CAP adds

Pre-commit hooks + layered rules

Mechanical gates at commit time: brief-quality checks, entity-schema validation, vendor-scrub, roundtrip integrity. Rules graduate from memory → heuristics → durable rule files. Where possible, every rule has a paired hook that can fail the commit.

.claude/hooks/ + .claude/rules/
It can't · by default

Challenge its own answers independently

A single model run is a single perspective. Asked to verify itself, it tends to confirm. Confident prose reads the same whether the underlying claim is tight or wrong.

→
CAP adds

Triad review + subagent dispatch

Consequential decisions dispatch to a three-perspective triad: challenger (adversarial), architect (structural), strategist (consequences). Parent agent is a filing clerk for their output, not an editor. Acceptance isn't validation — independent verification is.

triad pattern · subagent output contract
It can't · by default

Improve from being used

Model weights don't update from your conversations. Good patterns discovered this month dissipate by next. Fixes land on individual sessions — not on the underlying behavior.

→
CAP adds

Graduation pipeline · heuristic → rule

Analyst friction captured as feedback. Repeated firings crystallize into heuristic register entries. A triad review graduates proven heuristics to durable rule files (behavioral L2 or operating-model L3). Where mechanically enforceable, a paired hook follows. The rulebook improves without an author.

The honest framing

What this is, and what it isn't

It is a plumbing system that makes Claude behave consistently as your commercial analyst.

The layers above compose into a capability that knows your portfolio, follows your methodology, persists what it learns, and defends its answers against independent review. The model does the reasoning; the layers make that reasoning useful at a pharma company.

It isn't magic, and it isn't a better LLM.

The underlying model is the same one anyone can buy from Anthropic. What differs is the surrounding system — the memory, the rules, the graph, the enforcement, the graduation loop. Swap the model for a newer one tomorrow; the value stays in the layers. That's the point.

It is portable.

The domain layer is a single file. Replace it and the same capability re-points at a different industry — manufacturing, insurance, energy. Harness, behavioral, operating, role, hook, and KB plumbing all survive the swap. The agent is the product; the domain is config.
